Shinari grows by adding **providers**, not by patching the engine. Most
extensions are pure YAML; the rest is one Go file against the `sdk` package.
Engine internals (architecture, contracts, design principles) live in
[`DEVELOPERS.md`](https://github.com/shinari-dev/shinari/blob/main/DEVELOPERS.md)
in the repository.
